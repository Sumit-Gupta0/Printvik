================================================================================
PRINTVIK ARCHITECTURE DOCUMENTATION
================================================================================

This document serves as the high-level map for the PrintVik Business Operating System.

================================================================================
PART 1: BACKEND STRUCTURE (Node.js / Express)
================================================================================

[MODELS] - /backend/models/
--------------------------------------------------------------------------------
1.  User.js                : Stores Users, Operators, Riders, Admins. Includes Auth & Profile data.
2.  Order.js               : Central Order entity. Links User, Operator, Rider. Stores Status & Files.
3.  Product.js             : E-commerce products (Stationery, etc.). Includes Inventory & Variants.
4.  Address.js             : User delivery addresses.
5.  ServiceType.js         : Printing services (A4, A3, Color, B&W) configuration.
6.  DeliveryZone.js        : Polygon definitions for delivery areas and pricing.
7.  Coupon.js              : Discount codes and logic.
8.  Ledger.js              : Financial transactions (Credits/Debits) for all entities.
9.  PaymentSettlement.js   : Records of payouts made to Operators/Riders.
10. OperatorPayout.js      : Specific payout requests/records for Operators.
11. NotificationTemplate.js: SMS/Email/Push templates (Twilio/SendGrid/FCM).
12. NotificationLog.js     : History of all sent notifications.
13. AdminAlert.js          : In-app alerts for Admins (KYC, Low Stock, etc.).
14. GlobalSettings.js      : System-wide settings (Maintenance mode, etc.).
15. SystemConfig.js        : Dynamic configuration keys.
16. PricingConfig.js       : Base rates for printing and delivery.

[CONTROLLERS] - /backend/controllers/
--------------------------------------------------------------------------------
A. STANDARD CONTROLLERS
   - authController.js     : Register, Login, GetMe.
   - userController.js     : Profile, Address, KYC Upload.
   - orderController.js    : Create Order, Get My Orders, Track Order.
   - productController.js  : Browse Products, Search.
   - riderController.js    : Rider App API (Active Jobs, Update Status).
   - operatorController.js : Operator App API (Job Queue, Update Status).
   - paymentController.js  : Payment Gateway Integration (Razorpay/Stripe).
   - systemController.js   : Public config fetch.
   - deliveryZoneController: Zone checks.

B. ADMIN CONTROLLERS - /backend/controllers/admin/
   - adminAuthController.js   : Admin Login, PIN Management.
   - adminUserController.js   : Manage Users, Verify KYC, Wallet Ops, Operator Rates.
   - adminOrderController.js  : Manage Orders, Assign Riders, Refunds, File Verification.
   - adminContentController.js: CMS (Services, Zones, Coupons, Notifications).
   - adminStatsController.js  : Analytics, Dashboard Stats, Reports.

[SERVICES] - /backend/services/
--------------------------------------------------------------------------------
1.  NotificationService.js : Unified engine for sending SMS, Email, Push. Handles templating & logging.

[ROUTES] - /backend/routes/
--------------------------------------------------------------------------------
- /api/auth       -> authController
- /api/users      -> userController
- /api/orders     -> orderController
- /api/products   -> productController
- /api/delivery   -> riderController
- /api/operators  -> operatorController
- /api/admin      -> admin/*Controllers
- /api/payments   -> paymentController

PART 3: CRITICAL DATA FLOWS
================================================================================

[FLOW 1: ASSIGNING A RIDER]
--------------------------------------------------------------------------------
1.  **Frontend**: Admin clicks "Assign" on Order Card in `Orders.jsx`.
2.  **API Call**: POST `/api/admin/orders/assign` with `{ orderId, riderId }`.
3.  **Controller**: `adminOrderController.assignDelivery`.
    - Validates Rider status.
    - Updates Order status to 'out_for_delivery'.
    - Updates Rider's active count.
4.  **Service**: Calls `NotificationService.send('RIDER_ASSIGNED')`.
    - Sends Push Notification to Rider App.
5.  **Database**: Order & User documents updated.

[FLOW 2: ORDER COMPLETION & PAYOUT SPLIT]
--------------------------------------------------------------------------------
1.  **Trigger**: Rider marks order as "Delivered" (via Rider App) OR Admin updates status.
2.  **Controller**: `riderController.updateDeliveryStatus` OR `adminOrderController.updateOrder`.
3.  **Logic**:
    - Checks if Payment Status is 'Paid'.
    - Calculates Platform Fee (e.g., 10%).
    - Calculates Delivery Fee (Fixed/Distance).
    - Calculates Shop Share (Remainder).
4.  **Database**:
    - Creates `Ledger` entries:
      - CREDIT to Admin (Platform Fee).
      - CREDIT to Rider (Delivery Fee).
      - CREDIT to Operator (Shop Share).
    - Updates Order Status to 'completed'.
